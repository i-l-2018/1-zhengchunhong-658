git的工作原理
1.
在工作区修改文件，将修改后的文件保存到暂存区，提交（commit），将修改后的文件转存到本地仓库。通过push，将本地文件转存到远程仓库。
2.
在远程仓库进行的改动，通过fetch存储到本地仓库，再通过merge存储到工作区。

git与GitHub的区别
1.
git 是一个版本控制系统。是可以在电脑不联网的情况下，只在本地使用的一个版本管理工具，其作用就是可以更好的管理程序，比如：原来提交过的内容，以后虽然修改了，但是通过git这个工具，可以把原来提交的内容重现出来，这样可以将后来才意识到的一些错误的更改进行还原。git允许通过提交对一个系统（或一组）文件的历史进行注释。这些提交便是在给定时间点对系统做出的差异“快照”。这样就能知道自己在什么时侯对项目做了什么修改。 Git 采用了分布式版本库的方式，不必服务器端软件支持，使源代码的发布和交流极其方便。 Git 的速度很快，这对于大项目来说很重要，其最为出色的是它的合并跟踪能力。从git的内部工作机制来说，太过于晦涩难懂，但是随着开发的深入，Git 的正常使用都由一些友好的脚本命令来执行，使 Git 变得非常好用，即使是用来管理我们自己的开发项目，Git 都是一个友好，有力的工具。现在，越来越多的著名项目采用 Git 来管理项目开发。
2.

github 是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名gitHub。 作为开源代码库以及版本控制系统，Github拥有超过900万开发者用户。随着越来越多的应用程序转移到了云上，Github已经成为了管理软件开发以及发现已有代码的首选方法。 如前所述，作为一个分布式的版本控制系统，在Git中并不存在主库这样的概念，每一份复制出的库都可以独立使用，任何两个库之间的不一致之处都可以进行合并。 GitHub可以托管各种git库，并提供一个web界面， GitHub的独特卖点在于从另外一个项目进行分支的简易性。为一个项目贡献代码非常简单，在GitHub进行分支就像在进行交友一样，在社会关系图的节点中不断的连线。 GitHub项目本身自然而然的也在GitHub上进行托管，只不过在一个私 有的，公共视图不可见的库中。开源项目可以免费托管，但私有库则并不如此。在GitHub，用户可以十分轻易地找到海量的开源代码。
